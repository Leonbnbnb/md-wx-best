# 需求分析：面向微信公众号排版的通用 Markdown 渲染组件（更新版）

## 一、背景与目标

- 背景：内容团队以 Markdown 写作，发布到微信公众号时常出现样式丢失、代码块渲染差等问题。
- 目标：提供一个以 NPM 包形式分发的 Markdown 渲染组件，为微信公众号发布场景做排版与复制优化，可被其他项目集成使用。
- 成功标准：接入方传入 Markdown 文本后，组件可稳定渲染、切换主题、切换视图、复制为公众号支持的样式，且视觉一致、交互流畅。

## 二、术语

- 预览组件：接收 Markdown 文本并在页面中渲染的可视化组件。
- 顶部设置按钮：位于预览区域顶部、居中对齐的图标按钮集合，用于主题切换、视图切换、复制。
- 手机视图 / 桌面视图：模拟移动端和桌面端阅读时的排版呈现。
- 公众号支持的样式：被微信公众号编辑器正确解析与保存的内容结构与样式（允许的标签 + 内联样式）。

## 三、使用者与场景

- 使用者：前端工程师（接入组件）、内容运营/编辑（复制到公众号）。
- 场景：在 CMS/站点实时预览 Markdown；复制渲染结果到公众号；切换主题与视图检查阅读体验。

## 四、范围

- 在范围：Markdown 渲染预览、主题切换、代码块样式、视图切换、复制为公众号格式、设置按钮展示与导出、NPM 包分发。
- 不在范围：Markdown 编辑器、图床与图片上传、公众号平台接口对接、账号与权限体系。

## 五、功能性需求

### 1. 预览组件（实时渲染）
- 受控输入：传入 Markdown 字符串，内容变化实时更新。
- 覆盖常见语法：标题、段落、列表、引用、图片、链接、表格、代码块等。
- 发布习惯：字体、行高、间距、层级清晰。

### 2. 主题切换（5 个预设主题，默认第 1 个）
- 最新五个预设主题（与当前 UI/UX 保持一致）：
  1) 简约白：清爽留白，偏 Notion 风；
  2) 掘金酱紫：紫系强调，内容氛围突出；
  3) 樱花粉：柔和粉系，标题与链接粉色系强调；
  4) 科技蓝：冷色科技，信息密度与清晰度兼顾；
  5) 复古橙：衬线复古风，适合叙事与专栏。
- 要点：
  - 切换仅影响视觉（颜色、字号、间距、字体），不改变文档结构；
  - 标题、引用、列表、强调、链接等随主题显著变化；
  - 当前主题状态在设置按钮中清晰可识别。

### 3. 代码块样式（macOS 装饰 + 默认 GitHub Dark 风格）
- 代码块采用 macOS 风格容器（红黄绿三色圆点装饰）。
- 默认语法高亮视觉倾向 GitHub Dark。
- 复制为公众号格式时保留代码块结构与可读性（可降级移除装饰，不影响内容）。

### 4. 响应式设计（手机/桌面视图，默认手机）
- 提供手机与桌面两种预览模式，默认手机。
- 视图切换影响内容宽度、字号与间距等排版尺度。
- 与主题切换相互独立。

### 5. 一键复制为微信公众号支持的样式
- 输出仅使用公众号允许的标签与内联样式（白名单）。
- 标题、段落、列表、引用、表格、代码块等结构与关键样式保留。
- 移除不被支持的样式/属性，避免粘贴错乱；复制成功/失败可感知。

### 6. 顶部设置按钮（悬浮固定，居中图标化）
- 位于预览区域顶部居中，以图标形式展示主题、视图、复制。
- 内容滚动不影响按钮可见性。

### 7. 设置按钮显隐控制（默认显示）
- 可配置是否显示顶部设置按钮；隐藏不保留占位。

### 8. 导出按钮能力（自定义位置与排版）
- 将设置相关按钮导出，接入方可自定义摆放和组合（仅复制或仅主题）。
- 外置与内置按钮状态保持一致。

## 六、非功能性需求

- 可用性：数千字与多图/多表/多代码块下渲染流畅、交互响应 < 200ms。
- 一致性：不同主题/视图下结构与语义一致。
- 兼容性：现代浏览器环境；复制到公众号后保持预期样式。
- 可访问性：按钮可键盘达；复制反馈明确。
- 可配置性：主题、视图、装饰显隐、顶部设置显隐等可配置。
- 可移植性：以 NPM 包方式分发，类型与版本清晰。

## 七、交互与信息架构

- 顶部设置按钮：悬浮固定且居中；包含主题、视图、复制；默认显示，可关闭。
- 导出按钮：可在外部容器独立渲染，能力与状态一致。
- 预览区：渲染后的 Markdown；视图切换影响版心与字号，不影响主题色系选择。

## 八、对外输入与配置

- 必需输入：Markdown 文本（string）。
- 可选配置（示例）：
  - 默认主题（默认“简约白”）；
  - 默认视图（mobile/desktop，默认 mobile）；
  - 主题列表（内置 5 套，允许扩展/替换）；
  - 是否显示顶部设置按钮（默认 true）；
  - 是否显示代码块装饰（默认显示）；
  - 复制回调/提示配置。

## 九、错误与边界

- 空内容：显示空态；复制提示无内容。
- 超长文档：渲染与滚动流畅；长表格/长代码块不破版；粘贴后结构清晰。
- 不支持元素：复制阶段进行降级（保留结构、去除不支持样式）。

## 十、发布与交付

- 以 NPM 包方式交付；包含组件、主题、复制能力、使用说明与变更记录。

## 十一、验收标准（核心）

1) 实时渲染：输入变化即时更新，无明显卡顿与闪烁。  
2) 主题切换：五套主题，默认“简约白”；切换后标题/引用/列表/强调/链接风格明显变化且结构不变。  
3) 代码块样式：macOS 装饰 + GitHub Dark；复制后可读性良好。  
4) 响应式视图：默认手机；切换桌面后版心与字号有感知变化。  
5) 复制为公众号格式：粘贴至编辑器后整体排版与预览基本一致，无大面积样式丢失。  
6) 顶部设置按钮：悬浮固定、居中、图标化，点击可用。  
7) 显隐控制：可配置隐藏顶部设置按钮；显隐切换无版式抖动。  
8) 导出按钮：外置按钮可用，主题/视图/复制控制与内置一致，状态同步无冲突。
