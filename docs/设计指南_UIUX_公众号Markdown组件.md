# UI/UX 设计指南：微信公众号排版优化的 Markdown 渲染组件

## 1. 设计目标

- 面向微信公众号发布的 Markdown 预览与复制体验，确保“所见即所得”与“复制粘贴后风格可控”。
- 在预览中体现清晰的信息层级与舒适的阅读节奏；复制输出严格遵循公众号的样式限制（内联样式 + 白名单标签）。
- 提供 5 套主题，主题切换需显著改变标题、引用、列表、强调文字与链接的视觉风格。

## 2. 全局布局与动效

- 顶部设置区域
  - 悬浮固定在预览区域顶部、居中显示；图标化按钮：🌈 主题、📱/💻 视图、📋 复制。
  - 菜单（主题选择）在按钮下方展开；背景采用玻璃拟态（半透明 + 毛玻璃 + 轻边框）。
- 页面背景
  - 深色渐变基底 + 环境光晕，渲染技术氛围，但与预览内容的主题风格隔离。
  - 不随主题联动；内容风格切换集中在预览区。
- 动效与过渡
  - 交互控件采用 200–260ms 的缓动过渡（cubic-bezier(0.2, 0.8, 0.2, 1)）。
  - 主题切换引发颜色、字号、边框/背景变化时应平滑过渡（避免突兀闪烁）。

## 3. 主题系统（5 套预设）

主题切换需同时影响以下元素：
- H1–H6 字号/颜色/字体
- 段落正文颜色与行高
- 引用块背景与左侧边线颜色
- 列表标记颜色
- 强调文字（strong/em）颜色
- 链接颜色与悬停色（是否加下划线）
- 行内代码背景/边框/文字颜色
- 代码块容器（背景、边框、文字颜色）

预设主题（与 demo 对齐）：
- 简约白：清爽留白，偏 Notion 风
- 掘金酱紫：紫系强调，内容氛围突出
- 樱花粉：柔和粉系，标题与链接粉色强调
- 科技蓝：冷色科技，信息密度与清晰度兼顾
- 复古橙：衬线复古风，适合叙事与专栏

实现建议：使用统一命名的主题 token（示例）
- --t-surface：内容背景
- --t-text：正文文字
- --t-heading-font：标题字体族
- --t-heading：标题主色
- --t-strong / --t-em：强调文字色
-, --t-link / --t-link-hover / --t-link-decoration：链接颜色/悬停/是否下划线
- --t-quote-bg / --t-quote-border / --t-quote-text：引用块背景/边线/文字
- --t-list-marker：列表项标记颜色
- --t-inline-code-bg / --t-inline-code-border / --t-inline-code-text：行内代码
- --t-code-bg / --t-code-border / --t-code-text：代码块
- --t-body / --t-line / --t-h1..--t-h6：字号与行高

## 4. 视图模式（手机/桌面）

- 手机模式（默认）：模拟 iPhone 风格，内容宽度较窄、字号与行距更适配移动阅读。
- 桌面模式：更宽的版心与较大的标题字号，利于桌面端浏览。
- 视图切换仅影响内容区域的宽度与排版尺度，不改变主题色系。

## 5. Markdown 元素的视觉规范

- 标题（H1–H6）：
  - 标题间距：上大下小（如 margin-top 16px、margin-bottom 10px）。
  - 字号随层级递减，字体使用 --t-heading-font，颜色采用 --t-heading。
- 段落与正文：
  - 字号 --t-body，行高 --t-line，文字颜色 --t-text，字距适度（0.1–0.2px）。
- 强调文字：
  - strong 使用 --t-strong（更高对比度），em 使用 --t-em（倾斜或浅色强调）。
- 链接：
  - 常态使用 --t-link；悬停使用 --t-link-hover；下划线与否由 --t-link-decoration 控制。
- 引用块：
  - 背景 --t-quote-bg、左边线 --t-quote-border、文字 --t-quote-text；圆角适中（14px）。
- 列表：
  - 左内边距适中（约 22px），标记颜色使用 --t-list-marker；相邻项间距（6px）。
- 行内代码：
  - 使用 --t-inline-code-bg/border/text；圆角 8px、内边距 2px 6px；等宽字体。
- 代码块（macOS 容器 + GitHub Dark 外观）：
  - 顶栏装饰：红/黄/绿圆点（装饰，不参与复制输出）。
  - 容器：--t-code-bg/--t-code-border；正文文字使用 --t-code-text；等宽字体、行高 1.6–1.7。

## 6. 顶部设置区域（Toolbar）

- 固定定位于预览容器顶部，始终可见；居中对齐。
- 图标化按钮：
  - 主题（🌈）点击展开下拉菜单，显示主题名与渐变色块；当前主题标识清晰。
  - 视图（📱/💻）切换手机/桌面模式；按钮图标随状态变化。
  - 复制（📋）一键复制“公众号兼容 HTML”（内联样式），复制结果弹出轻提示。

## 7. 复制为公众号格式（设计约束）

- 输出仅允许白名单标签：p、br、div、span、h1–h6、strong、em、u、s、blockquote、ul/ol/li、pre/code、img、a、table/tr/td/th（可选降级）。
- 样式仅使用内联 style；不包含 class/id/事件属性；不使用 position/float/flex/grid/transform 等高风险属性（跨端不稳定）。
- 链接：可配置策略（默认保守）——保留文本样式，可选允许 mp.weixin.qq.com 域名保留 href。
- 图片：仅 https；保留 alt；宽图限制最大宽度，避免破版。
- 代码块：强调可读性；高亮 token 在被过滤时降级为单色等宽文本。

## 8. 图标与可访问性

- 图标优先系统 emoji（🌈、📱、💻、📋），避免额外资源；必要时可替换为 SVG。
- 提供 ARIA 与键盘支持：
  - toolbar role="toolbar"；按钮可聚焦，Enter/Space 触发；菜单 role="listbox"；Esc 关闭。
  - 提示信息 role="status"、aria-live="polite"。

## 9. 动画与微交互

- 悬停上浮 1–2px，点击压下 1%（scale 0.99）；下拉菜单出现使用 pop 动画（180ms）。
- 滚动条使用浅色圆角样式，弱化视觉干扰。

## 10. 命名与资源规范

- 主题 ID：t-white、t-juejin、t-wechat-dark（已改为“樱花粉”）、t-tech-blue、t-vintage-orange。
- 主题 token：统一使用 --t-* 命名，便于渲染与复制时的映射。
- 资源：不引入外部字体；尽量使用系统字体组合；图标与插图在复制链路中需可降级。

## 11. 验收清单（UI/UX）

- 主题切换：标题/引用/列表/强调/链接风格显著变化；过渡平滑。
- 视图切换：版心宽度、字号与间距随之调整；不改变主题色系。
- 代码块：macOS 顶栏装饰存在，GitHub Dark 可读性好，长代码滚动顺畅。
- 复制：粘贴到公众号编辑器后结构与关键样式保留，无明显错乱；代码块可读。
