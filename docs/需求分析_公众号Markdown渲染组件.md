# 需求分析：面向微信公众号排版的通用 Markdown 渲染组件

## 一、背景与目标

- 背景：内容团队大量产出 Markdown 文章，并需要在微信公众号发布。原生复制到公众号编辑器常出现样式丢失、代码块不可读、排版不一致等问题。
- 目标：提供一个以 NPM 包形式分发的“通用 Markdown 渲染组件”，为微信公众号发布场景优化排版与复制体验，可被其它项目集成使用。
- 成功标准：接入方传入 Markdown 文本后，组件可稳定渲染、切换主题、切换视图、复制为公众号支持的样式，并保持良好可用性与一致的视觉输出。

## 二、术语定义

- 预览组件：接收 Markdown 文本并在页面中渲染的可视化组件。
- 顶部设置按钮：位于预览区域顶部、居中对齐的图标按钮集合，用于主题切换、视图切换、复制等操作。
- 手机视图 / 桌面视图：用于模拟在移动端或桌面端阅读时的排版呈现。
- 公众号支持的样式：可被微信公众号编辑器正确解析与保存的内容结构与样式（以其允许的 HTML 标签和行内样式为准）。

## 三、使用者与场景

- 使用者：前端工程师（接入组件）、内容运营/编辑（使用复制功能将内容粘贴至公众号）。
- 场景：
  - 在文档平台或 CMS 中实时预览 Markdown 并优化排版；
  - 复制渲染结果到微信公众号编辑器；
  - 根据不同内容风格切换预设主题；
  - 在手机/桌面视图间切换以校验阅读体验。

## 四、范围与不在范围

- 在范围：Markdown 渲染预览、主题切换、代码块装饰与样式、手机/桌面视图切换、复制为公众号支持的样式、设置按钮展示与导出能力、作为 NPM 包对外提供。
- 不在范围：Markdown 内容编辑器本身、图片上传/图床管理、与公众号平台的自动化接口对接、权限与账号体系。

## 五、功能性需求

### 1. 预览组件（实时渲染）
- 需求：组件接收外部提供的 Markdown 文本内容，并在内容变化时实时更新渲染结果。
- 要点：
  - 以“受控输入”的方式工作，输入变化即触发重渲染；
  - 支持标准 Markdown 语法（标题、段落、列表、引用、图片、链接、表格、代码块等）；
  - 渲染完成后排版满足公众号发布阅读习惯（字体大小、行高、段落间距、标题层级清晰）。
- 验收：输入文本在 300ms 内可见更新；常见 Markdown 元素均能正确渲染且结构清晰。

### 2. 主题切换（5 个预设主题，默认第 1 个）
- 需求：内置五套排版主题风格，并支持在它们之间切换；默认展示第一套主题。
- 预设主题（命名与意图）：
  1) WeChat Classic（经典微信）：接近公众号通用阅读风格，简洁、中性；
  2) Fresh Minimal（清新简约）：空间感更强，留白更足，适合教程/长文；
  3) Elegant Serif（典雅衬线）：更具书卷气，适合随笔/论文摘录；
  4) Comfort Green（护眼墨绿）：柔和配色，长时间阅读更舒适；
  5) Dark Night（暗夜深邃）：深色背景，弱光环境阅读友好。
- 要点：
  - 切换主题不改变文档结构，仅影响视觉呈现（颜色、间距、字号等）；
  - 主题之间的代码块可读性保持一致；
  - 主题名称与当前激活状态在设置按钮中可识别。
- 验收：切换后 200ms 内生效；不同主题下标题、正文、链接、引用、表格等元素有明确差异且可读。

### 3. 代码块样式（Mac 风格装饰 + 默认 github-dark）
- 需求：代码块展示 Mac 风格装饰图标（如三色圆点窗控元素），默认代码高亮风格为“github-dark”。
- 要点：
  - 装饰图标为视觉元素，不影响代码内容复制；
  - 在复制为公众号样式时，保留代码块结构和基本可读性；
  - 允许在需求层面配置是否显示装饰（可由接入方决策）。
- 验收：代码块顶部出现一致的 Mac 风格装饰；多语言代码在默认风格下具备清晰对比度与可读性。

### 4. 响应式设计（手机/桌面视图切换，默认手机）
- 需求：提供“手机视图”和“桌面视图”两种预览模式，默认手机视图。
- 要点：
  - 手机视图：更紧凑的字号与边距、更短的行长；
  - 桌面视图：更宽内容区与更长行宽、更适合大屏浏览；
  - 视图切换与主题切换相互独立。
- 验收：切换后版心宽度、字号与间距等随视图发生可感知变化；默认初始为手机视图。

### 5. 复制为微信公众号支持的样式
- 需求：一键复制预览区域内容为“公众号可粘贴且保留样式”的格式。
- 要点：
  - 复制内容应仅包含被公众号编辑器允许的标签与行内样式；
  - 标题、段落、列表、引用、图片、表格、代码块等保留结构与必要样式；
  - 移除不被支持的样式/属性，避免粘贴后出现样式错乱；
  - 代码块以可读性优先，保持背景/边框/等宽字体等关键视觉；
  - 复制成功与否提供用户可感知的结果反馈。
- 验收：将内容粘贴至公众号编辑器后，整体排版与预览基本一致；不出现大段样式丢失或结构混乱。

### 6. 顶部设置按钮（居中且不随内容滚动）
- 需求：设置按钮在预览区域顶部、居中对齐展示，以图标形式呈现；内容滚动时按钮保持固定位置。
- 要点：
  - 常用操作：主题切换、视图切换、复制；
  - 视觉样式与各主题保持协调；
  - 在内容很长时，按钮不被内容遮挡。
- 验收：滚动预览内容时，顶部设置按钮始终可见且居中；点击操作可用。

### 7. 可控制显示顶部设置按钮（默认显示）
- 需求：接入方可控制是否显示顶部设置按钮，默认显示。
- 要点：隐藏时不保留占位，不影响内容布局；显示时遵循“顶部居中且固定”的规则。
- 验收：切换开关后，按钮显/隐即时生效，布局无异常位移。

### 8. 设置相关按钮支持导出（自定义位置与排版）
- 需求：将设置相关按钮能力导出，允许接入方自行在页面任意位置摆放并自定义排列方式。
- 要点：
  - 可独立渲染这些控制项，支持组合或单独使用（如仅复制按钮）；
  - 导出的按钮与内置按钮具备一致的行为与状态联动；
  - 在自定义位置时，不破坏预览组件的其他功能性。
- 验收：接入方可在外部渲染按钮，并正常控制主题、视图与复制，状态同步无延迟与冲突。

## 六、非功能性需求

- 可用性：对常见 Markdown 文档（数千字、含多图/多表/多代码块）渲染流畅、滚动顺滑；常规交互响应不超过 200ms。
- 一致性：不同主题/视图下的基础排版规则一致，切换不改变正文结构。
- 兼容性：满足现代浏览器环境，移动端与桌面端显示一致性良好；复制到公众号编辑器后可保留预期样式。
- 可访问性：设置按钮具备清晰的可点击区域与键盘可达性；复制成功/失败具备状态提示。
- 可配置性：主题、是否显示装饰图标、是否显示顶部设置、默认视图等可作为对外配置。
- 可移植性：作为 NPM 包可被多项目安装使用，提供清晰版本号与变更记录（发布规范见下）。

## 七、交互与信息架构

- 顶部设置按钮（内置）：
  - 图标按钮集合，包含：主题切换、视图切换、复制；
  - 居中对齐，固定在预览区域顶部，滚动不动；
  - Hover/Focus 状态清晰、点击态反馈明确；
  - 默认展示；接入方可关闭。
- 导出按钮（外置）：
  - 暴露相同能力与状态，允许接入方放置于自定义容器；
  - 与内置按钮互斥或并存均可，由接入方选择；
  - 当外置按钮存在时，内置按钮可独立控制是否仍显示。
- 预览区：
  - 渲染后的 Markdown 内容区域；
  - 视图切换影响排版宽度与字号，但不影响主题色系选择。

## 八、对外输入与配置（需求层级描述）

- 必需输入：Markdown 文本内容（字符串）。
- 可选配置（示例维度）：
  - 默认主题键值（默认“ WeChat Classic”）；
  - 主题列表（内置 5 套，允许接入方扩展/替换）；
  - 默认视图（mobile/desktop，默认 mobile）；
  - 是否显示顶部设置按钮（默认 true）；
  - 是否显示代码块装饰图标（默认显示）；
  - 复制后的回调/结果反馈开关（用于业务方提示）。

## 九、错误与边界

- 空内容：显示空态文案或空白，操作按钮除复制外仍可用；复制时给出无内容提示。
- 超长文档：渲染与滚动保持流畅，长表格/长代码块不破版；复制后在公众号编辑器中仍保持结构清晰。
- 不支持元素：遇到非标准或公众号不支持的样式时，在复制阶段进行降级处理（保留结构、去除不支持的样式）。

## 十、发布与交付（需求层级）

- 交付形式：以 NPM 包方式发布，供其它项目安装使用；提供基础使用说明与变更记录。
- 包含内容：
  - 主组件（预览）、内置顶部设置按钮、导出按钮能力；
  - 五套内置主题；
  - 复制为公众号样式的能力；
  - 使用与配置说明文档。

## 十一、验收标准（对应核心需求）

1) 预览实时渲染：输入变化后内容即时更新，无明显闪烁或卡顿。
2) 主题切换：五套主题任选其一，默认第 1 套；切换后视觉差异明显，结构保持一致。
3) 代码块样式：Mac 风格装饰可见，默认高亮风格具备良好对比与可读性；复制后代码区域仍清晰。
4) 响应式视图：默认手机视图，切换桌面视图后版心与字号有感知变化。
5) 复制为公众号样式：粘贴到公众号编辑器后版式基本一致，常见元素表现正确，无大面积样式丢失。
6) 顶部设置按钮：顶部居中且固定，滚动内容不影响其可见与可点；图标清晰。
7) 显隐控制：通过配置可隐藏顶部设置按钮；显/隐切换无版式抖动。
8) 导出按钮：外置按钮在任意位置可用，能控制主题/视图/复制，且与内置按钮状态同步。

## 十二、里程碑（需求拆分）

- M1：基础渲染 + 手机/桌面视图切换 + 顶部设置按钮框架（默认显示、居中固定）。
- M2：主题系统（5 套预设 + 默认主题）与代码块装饰样式。
- M3：复制为公众号支持的样式（规则与降级策略）+ 成功/失败反馈。
- M4：导出设置按钮能力（外置控制与状态同步）。
- M5：完善文档与发布规范，完成对接项目集成验证。
